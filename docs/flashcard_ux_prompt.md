# 闪卡学习小程序 — UX 设计稿 Prompt

---

## 项目背景

这是一个微信小程序闪卡学习应用。用户手动创建卡组和卡片，
通过翻转卡片+自评打分进行复习，系统基于间隔重复算法自动安排下次复习时间。

---

## 技术栈

- 框架：Taro + React（可直接编译为微信小程序）
- 样式：WXSS / Taro 内置样式方案，单位用 rpx
- 设备基准：iPhone 15 Pro，390px / 750rpx 宽度
- 遵循微信小程序 UI 规范（安全区域、导航栏高度等）
- 不需要桌面端适配

---

## 设计风格

- 风格：  Warm Progress 
- 字体：微信小程序默认字体体系

---

## 页面清单（共 5 页）

### Page 1 — 首页 / 仪表盘

**展示内容：**
- 今日待复习卡片数量（醒目大数字）
- 当前学习连续天数（Streak）
- 各卡组掌握率概览（进度条列表，最多显示 3 个，更多点击跳转）

**操作入口：**
- 「开始今日复习」主按钮
- 「新建卡组」快捷入口
- 底部 TabBar 导航

---

### Page 2 — 卡组列表

**展示内容：**
- 所有卡组卡片（卡组名、卡片数量、掌握率、下次复习时间）
- 空状态引导（没有卡组时显示引导插图+创建按钮）

**操作：**
- 右上角「+」新建卡组（弹出命名输入框）
- 点击卡组进入卡片列表
- 左滑卡组显示：编辑名称 / 删除

---

### Page 3 — 卡片列表（进入某个卡组后）

**展示内容：**
- 顶部：卡组名称 + 卡片总数 + 掌握率
- 卡片列表（正面问题预览、掌握状态标签：掌握/模糊/不会/未学）

**操作：**
- 右上角「+」新增卡片（跳转到编辑页，输入正面/背面内容）
- 点击卡片进入编辑
- 左滑卡片：删除
- 底部「开始复习」悬浮按钮

---

### Page 4 — 复习模式

**展示内容：**
- 顶部进度条 + 计数（第 X / 共 Y 张）
- 卡片主体（支持翻转动画）
  - 正面：问题
  - 背面：答案
- 翻转前：「查看答案」按钮
- 翻转后：自评打分按钮组（不会 / 模糊 / 掌握）

**操作：**
- 点击卡片或按钮翻转（CSS 3D flip 动画）
- 选择掌握程度后自动进入下一张
- 顶部可切换顺序 / 随机模式
- 复习完成后显示本轮总结页（掌握/模糊/不会各几张）

---

### Page 5 — 学习统计

**展示内容：**
- 近 7 天 / 30 天每日复习卡片数折线图
- 各卡组掌握率横向柱状图
- 连续学习天数日历热力图（类似 GitHub Contribution）

**操作：**
- 切换时间范围（7天 / 30天）

## 导航结构

```
底部 TabBar（微信小程序原生 tabBar）：
[🏠 首页]  [📦 卡组]  [📊 统计]

页面跳转：
- 卡组列表 → 卡片列表：wx.navigateTo
- 卡片列表 → 复习模式：wx.navigateTo
- TabBar 切换：wx.switchTab
```

---

## 设计规范（统一设计系统）

```
圆角：卡片 16px，按钮 12px，输入框 10px
阴影：卡片 0 4px 20px rgba(0,0,0,0.08)
间距：页面内边距 32rpx（两侧）
安全区域：底部留出 iPhone 底部 Home Indicator 空间
动画：卡片翻转 0.4s ease，页面切换用小程序默认 slide
掌握状态颜色：
  掌握 → 绿色 #34C759
  模糊 → 橙色 #FF9500
  不会 → 红色 #FF3B30
  未学 → 灰色 #C7C7CC
```

## 数据结构参考

```typescript
// 卡组
interface Deck {
  id: string
  name: string
  createdAt: number
  cards: Card[]
}

// 卡片
interface Card {
  id: string
  front: string       // 问题
  back: string        // 答案
  status: 'new' | 'learning' | 'review'
  ease: number        // SM-2 难度系数，初始 2.5
  interval: number    // 下次复习间隔（天）
  nextReview: number  // 下次复习时间戳
  repetitions: number // 已复习次数
}
```
